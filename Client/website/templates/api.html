Introduction

HelloString is a RESTful API that provides the ability to perform natural language processing on any given text, by providing access to open-source tools.

This web application doesn't run on a specific domain. This means that it's URL changes consistently. We will be refering to the current domain of the page as www.currentdomain.com.

In general, the API can be accessed through different endpoints - that are all described on this page - width the following, base URL: 

                            www.currentdomain.com:8001/api/*

Important notice: For every of the APIs methods, the text inputed must be follow a URL-encoded format. For more information, please visit this link.




Tokenization

Text tokenization is available by calling the API through the following endpoint: 

                                www.currentdomain.com/api/tokenize?text='input_string'

Tokenization is a basic NLP task that accepts a string as an input and returns tokens (words or sentences) in the form of an array of strings.

Inputs:

  • text: (Type: String *) The input string that needs to be tokenized

Outputs:

{
'sentence_tokens': [--Array of strings--], 
'word_tokens': [--Array of strings--]
}
The 'sentence_tokens' contains an array of all the sentences found in the input string while the 'word_tokens' contains an array of all the words found in the input string.




Word/Character Frequency

Word/character frequency detection is available by calling the API through the following endpoint: 

                                www.currentdomain.com/api/frequency?text='input_string'&level='level'

Inputs:

  • text: (Type: String *) The input string that needs to be analyzed
  • level: (Type: String, Choices: [char, word (default)], Optional) Frequency analysis on character/word level

Outputs:

{
'word_frequency': {JSON structure}
}
The 'word_frequency' is a JSON structured object containing the characters/words as keys and their frequency in the input text as a value.




Stopword removal

Stopword removal is available by calling the API through the following endpoint: 

                                www.currentdomain.com/api/stopwords?text='input_string'&sw_list=[--List of strings--]

Inputs:

  • text: (Type: String *) The input string that needs to be analyzed
  • sw_list: (Type: Array of Strings *, Optional) Array containing user-defined stopwords - overwrites default stopword list. For the default stopword list, visit the following link.

Outputs:

{
'stopwords': String
}
The 'stopwords' contains the input string, stripped of it's stopwords.




Stemming

Stemming is available by calling the API through the following endpoint: 

                                www.currentdomain.com/api/stemming?text='input_string'&stemmer='stemmer'

Inputs:

  • text: (Type: String *) The input string that needs to be stemmed
  • stemmer: (Type: String, Choices: [snowball, porter (default)], Optional) Stemming method used

Outputs:

{
'stemmed_string': String
}
The 'stemmed_string' contains the input string, after being stemmed through one of the above methods.

Sentiment Analysis

Sentiment analysis is available by calling the API through the following endpoint: 

                                www.currentdomain.com/api/sentiment?text='input_string'&auth='auth_key'&cont='cont'&method='method'

Inputs:

  • text: (Type: String *) The input string that needs to be lemmatized
  • auth: (Type: String ) The authentication key given to every verified account (find it under the "Settings" page)
  • cont: (Type: String, Choices: [no, yes (default)], Optional) The output should be continuous or categorical (yes or no)
  • method: (Type: String, Choices: [textblob, vader (default)], Optional) The model used for sentiment analysis

Outputs:

{
'sentiment': Float
}
The 'sentiment' represents a continuous (-1 to 1) or a discrete value (-1, 0 or 1) of the sentiment score. Negative scores represent a negative sentiment (-1 meaning most negative), positive scores represent a positive sentiment (1 meaning most positive) while scores close to 0 inticate neither positive or negative sentiment (neutral).




Text Aggressiveness

Text Aggressiveness is available by calling the API through the following endpoint: 

                                www.currentdomain.com/api/aggressiveness?text='input_string'&auth='auth_key'&cont='cont'

Inputs:

  • text: (Type: String *) The input string that needs to be lemmatized
  • auth: (Type: String ) The authentication key given to every verified account (find it under the "Settings" page)
  • cont: (Type: String, Choices: [no, yes (default)], Optional) The output should be continuous or categorical (yes or no)

Outputs:

{
'aggressiveness': Float
}
The 'aggressiveness' represents a continuous (0 to 1) or a discrete value (0 or 1) of the sentiment score. Scores closer to 1 represent a highly aggressive content (1 meaning most aggressive content) while scores closer to 0 inticate less aggressive content (0 meaning no aggressive content).




Error Codes

Error code 400 is usually refering to wrong parameters that are passed during a request.

For example, a 400 error code could be returned with the reason "No text argument", indicating that no "text" parameter is passed when needed.0


Error code 401 is refering to missing/invalid authentication. It's refering to the 'auth' parameter of some requests. The auth key is provided to every verified user, and can be found at the Settings page.
